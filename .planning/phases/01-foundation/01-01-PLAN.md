# Plan 01-01: Database, Seed Data, Supabase Helpers & Types

## Objective

Set up the complete Supabase data layer: database schema with RLS, seed data for 15 stays/5 collections/20 reviews, server + client Supabase helpers, and generated TypeScript types. This is the data foundation every subsequent feature builds on.

## Context

- **Stack**: Next.js 16, Supabase (local via `supabase start`), TypeScript, pnpm
- **Env vars**: Use `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` (NOT ANON_KEY — per project memory)
- **Auth pattern**: Use `proxy.ts` (NOT middleware.ts) and `getClaims()` (NOT getUser()) — per project memory. Proxy setup deferred to Phase 3, but helpers should be compatible.
- **Types**: Use `supabase gen types` for DB types, NOT a central types file. Colocate component types with components.
- **Pricing**: Store prices in cents (integer) to avoid floating-point bugs
- **Booking integrity**: DB-level date exclusion constraint (daterange with `&&` operator) to prevent double-booking
- **Vibes**: adventure, culture, disconnect, celebration
- **Travel types**: solo, duo, family, group

<task name="Create database migration with all tables and RLS" type="auto">
  <name>Create database migration with all tables and RLS</name>
  <files>supabase/migrations/00001_create_tables.sql, supabase/config.toml</files>
  <action>
    1. Initialize Supabase in the project if not already done: `pnpm dlx supabase init` (creates supabase/ directory with config.toml)

    2. Create migration file `supabase/migrations/00001_create_tables.sql` with the following tables:

    **profiles** (synced from auth.users):
    - `id` UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE
    - `email` TEXT NOT NULL
    - `full_name` TEXT
    - `avatar_url` TEXT
    - `created_at` TIMESTAMPTZ DEFAULT now()
    - `updated_at` TIMESTAMPTZ DEFAULT now()

    **stays**:
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `title` TEXT NOT NULL
    - `slug` TEXT UNIQUE NOT NULL
    - `description` TEXT NOT NULL
    - `location` TEXT NOT NULL
    - `price_per_night` INTEGER NOT NULL (cents)
    - `cleaning_fee` INTEGER NOT NULL DEFAULT 0 (cents)
    - `service_fee` INTEGER NOT NULL DEFAULT 0 (cents)
    - `max_guests` INTEGER NOT NULL DEFAULT 2
    - `type` TEXT NOT NULL CHECK (type IN ('treehouse', 'cabin', 'glamping', 'houseboat', 'yurt'))
    - `vibe` TEXT NOT NULL CHECK (vibe IN ('adventure', 'culture', 'disconnect', 'celebration'))
    - `travel_type` TEXT NOT NULL CHECK (travel_type IN ('solo', 'duo', 'family', 'group'))
    - `amenities` TEXT[] DEFAULT '{}'
    - `images` TEXT[] NOT NULL (array of Unsplash URLs)
    - `created_at` TIMESTAMPTZ DEFAULT now()
    - `updated_at` TIMESTAMPTZ DEFAULT now()

    **collections**:
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `title` TEXT NOT NULL
    - `slug` TEXT UNIQUE NOT NULL
    - `description` TEXT
    - `created_at` TIMESTAMPTZ DEFAULT now()

    **collection_stays** (junction table):
    - `collection_id` UUID REFERENCES collections(id) ON DELETE CASCADE
    - `stay_id` UUID REFERENCES stays(id) ON DELETE CASCADE
    - `position` INTEGER NOT NULL DEFAULT 0
    - PRIMARY KEY (collection_id, stay_id)

    **reviews**:
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `stay_id` UUID NOT NULL REFERENCES stays(id) ON DELETE CASCADE
    - `user_id` UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
    - `rating` INTEGER NOT NULL CHECK (rating BETWEEN 1 AND 5)
    - `comment` TEXT
    - `created_at` TIMESTAMPTZ DEFAULT now()
    - UNIQUE(stay_id, user_id) — one review per user per stay

    **bookings**:
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `stay_id` UUID NOT NULL REFERENCES stays(id) ON DELETE CASCADE
    - `user_id` UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
    - `check_in` DATE NOT NULL
    - `check_out` DATE NOT NULL
    - `guests` INTEGER NOT NULL
    - `total_price` INTEGER NOT NULL (cents)
    - `contact_name` TEXT NOT NULL
    - `contact_email` TEXT NOT NULL
    - `contact_phone` TEXT
    - `status` TEXT NOT NULL DEFAULT 'confirmed' CHECK (status IN ('confirmed', 'cancelled'))
    - `created_at` TIMESTAMPTZ DEFAULT now()
    - CHECK (check_out > check_in)
    - EXCLUDE USING gist (stay_id WITH =, daterange(check_in, check_out) WITH &&) — prevents double-booking at DB level. Requires btree_gist extension.

    **Enable btree_gist**: `CREATE EXTENSION IF NOT EXISTS btree_gist;` at top of migration.

    **RLS policies** (enable RLS on ALL tables):
    - `stays`: SELECT for everyone (anon + authenticated), no INSERT/UPDATE/DELETE via client
    - `collections`: SELECT for everyone
    - `collection_stays`: SELECT for everyone
    - `reviews`: SELECT for everyone, INSERT for authenticated (user_id = auth.uid()), UPDATE/DELETE for own reviews
    - `bookings`: SELECT for own bookings (user_id = auth.uid()), INSERT for authenticated
    - `profiles`: SELECT for everyone, UPDATE for own profile (id = auth.uid())

    **Trigger**: Create a trigger function `handle_new_user()` that inserts a profile row when a new auth.users row is created (copies id, email, raw_user_meta_data->full_name, raw_user_meta_data->avatar_url).

    3. Verify config.toml has reasonable defaults (project_id can be "lateral-challenge")
  </action>
  <verify>
    Run `supabase db reset` (which applies migrations + seed). Check that all 6 tables exist:
    ```bash
    supabase db reset
    ```
    If btree_gist extension causes issues with local setup, verify the migration SQL is syntactically valid by reviewing it.
  </verify>
  <done>Migration file exists with all 6 tables, RLS policies, btree_gist extension, and booking exclusion constraint</done>
</task>

<task name="Install Supabase packages, create client helpers, generate types" type="auto">
  <name>Install Supabase packages, create client helpers, generate types</name>
  <files>lib/supabase/server.ts, lib/supabase/client.ts, lib/supabase/types.ts, package.json</files>
  <action>
    1. Install Supabase packages:
    ```bash
    pnpm add @supabase/supabase-js @supabase/ssr
    ```

    2. Create `lib/supabase/server.ts` — server-side client for RSC and Server Actions:
    ```typescript
    import { createServerClient } from '@supabase/ssr'
    import { cookies } from 'next/headers'
    import type { Database } from './types'

    export async function createClient() {
      const cookieStore = await cookies()

      return createServerClient<Database>(
        process.env.NEXT_PUBLIC_SUPABASE_URL!,
        process.env.NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY!,
        {
          cookies: {
            getAll() {
              return cookieStore.getAll()
            },
            setAll(cookiesToSet) {
              try {
                cookiesToSet.forEach(({ name, value, options }) =>
                  cookieStore.set(name, value, options)
                )
              } catch {
                // Called from Server Component — ignore
              }
            },
          },
        }
      )
    }
    ```

    3. Create `lib/supabase/client.ts` — browser-side client for Client Components:
    ```typescript
    import { createBrowserClient } from '@supabase/ssr'
    import type { Database } from './types'

    export function createClient() {
      return createBrowserClient<Database>(
        process.env.NEXT_PUBLIC_SUPABASE_URL!,
        process.env.NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY!
      )
    }
    ```

    4. Generate types from the running local Supabase instance:
    ```bash
    pnpm dlx supabase gen types typescript --local > lib/supabase/types.ts
    ```
    If Supabase isn't running, create a placeholder `lib/supabase/types.ts` that exports a `Database` type with the correct table structure matching the migration. Add a comment: `// Generated by: pnpm dlx supabase gen types typescript --local > lib/supabase/types.ts`

    5. Create `.env.local` with placeholder values:
    ```
    NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
    NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=your-anon-key-from-supabase-start
    ```

    6. Add `.env.local` to `.gitignore` if not already there.

    IMPORTANT: Use `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` NOT `NEXT_PUBLIC_SUPABASE_ANON_KEY` — per project memory.
