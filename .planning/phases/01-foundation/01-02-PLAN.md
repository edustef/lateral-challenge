# Plan 01-02: Warm Organic Theme & Design System Setup

## Objective

Configure the visual foundation to match the Pencil design file: CSS custom properties for the warm organic palette, Tailwind v4 theme integration, proper font setup (Fraunces for headings, Inter for body, IBM Plex Mono for prices), and shadcn/ui initialization. After this plan, every component can use the design tokens.

## Context

- **Design source**: `lateral-design.pen` — variables extracted directly from Pencil file
- **Key colors from Pencil design**:
  - `--bg-page`: #FAF8F5 (warm cream page background)
  - `--bg-card`: #FFFFFF (white cards)
  - `--bg-muted`: #F0EDE8 (muted background)
  - `--bg-surface`: #F5F3EF (surface background)
  - `--warm-accent`: #C4956A (warm terracotta accent)
  - `--warm-accent-tint`: #C4956A20 (warm accent with 12% opacity)
  - `--accent` / `--primary` from design: green #7C9082 / orange #FF8400
  - `--text-primary`: #2D2D2D, `--text-body`: #5A5A5A, `--text-secondary`: #8A8A8A, `--text-muted`: #ADADAD
  - `--border`: #E8E4DF, `--border-subtle`: #F0EDE8
  - `--success`: #22C55E, `--error`: #E55B5B
- **Fonts from Pencil design**:
  - Headings: Fraunces (serif, used for logo + titles, fontSize 16-24, weight 500)
  - Body: Inter (sans-serif, fontSize 13-14)
  - Prices: IBM Plex Mono (monospace, fontSize 14, weight 500)
- **Radii from Pencil design**:
  - `--radius-card`: 20px, `--radius-button`: 24px, `--radius-input`: 28px, `--radius-badge`: 16px, `--radius-pill`: 999px, `--radius-small`: 12px
- **Stack**: Tailwind v4 (already installed), shadcn/ui (to be added)
- No dark mode needed for v1 — the design file has dark mode tokens but we focus on light mode only

## Tasks

<task name="Install shadcn/ui and configure fonts" type="auto">
  <name>Install shadcn/ui and configure fonts</name>
  <files>package.json, lib/fonts.ts, app/layout.tsx</files>
  <action>
    1. Initialize shadcn/ui:
    ```bash
    pnpm dlx shadcn@latest init
    ```
    When prompted: use default style, CSS variables = yes, base color = neutral, globals.css path = app/globals.css. Accept defaults — we will override the CSS variables in the next task.

    2. Create `lib/fonts.ts` to centralize font configuration:
    ```typescript
    import { Fraunces, Inter, IBM_Plex_Mono } from 'next/font/google'

    export const fraunces = Fraunces({
      subsets: ['latin'],
      variable: '--font-fraunces',
      display: 'swap',
    })

    export const inter = Inter({
      subsets: ['latin'],
      variable: '--font-inter',
      display: 'swap',
    })

    export const ibmPlexMono = IBM_Plex_Mono({
      subsets: ['latin'],
      weight: ['400', '500'],
      variable: '--font-ibm-plex-mono',
      display: 'swap',
    })
    ```

    3. Update `app/layout.tsx`:
    - Import fonts from `@/lib/fonts`
    - Remove Geist font imports (replace with our fonts)
    - Apply font CSS variables to `<body>`: `${fraunces.variable} ${inter.variable} ${ibmPlexMono.variable}`
    - Set `font-sans` default to Inter via the CSS variable
    - Update metadata: title = "Lateral — Unique Stays", description = "Discover unique stays through a vibe-first experience"
    - Keep `antialiased` class
  </action>
  <verify>
    ```bash
    pnpm build
    ```
    Build succeeds. Check that layout.tsx imports the 3 fonts and applies their CSS variables.
  </verify>
  <done>Three Google Fonts configured (Fraunces, Inter, IBM Plex Mono) and applied to root layout</done>
</task>

<task name="Configure CSS custom properties and Tailwind v4 theme" type="auto">
  <name>Configure CSS custom properties and Tailwind v4 theme</name>
  <files>app/globals.css</files>
  <action>
    Replace the contents of `app/globals.css` with the full design system. This must work with Tailwind v4's `@theme` directive.

    Structure of globals.css:

    ```css
    @import "tailwindcss";

    /* ===== Design Tokens from Pencil Design ===== */
    :root {
      /* Page backgrounds */
      --bg-page: #FAF8F5;
      --bg-card: #FFFFFF;
      --bg-muted: #F0EDE8;
      --bg-surface: #F5F3EF;

      /* Accents */
      --warm-accent: #C4956A;
      --warm-accent-tint: #C4956A20;
      --accent: #7C9082;
      --accent-tint: #7C908226;
      --primary: #FF8400;

      /* Text */
      --text-primary: #2D2D2D;
      --text-body: #5A5A5A;
      --text-secondary: #8A8A8A;
      --text-muted: #ADADAD;

      /* Borders */
      --border: #E8E4DF;
      --border-subtle: #F0EDE8;

      /* Semantic */
      --success: #22C55E;
      --error: #E55B5B;
      --destructive: #D93C15;

      /* Radii */
      --radius-card: 20px;
      --radius-button: 24px;
      --radius-input: 28px;
      --radius-badge: 16px;
      --radius-pill: 999px;
      --radius-small: 12px;
      --radius-m: 16px;

      /* Fonts */
      --font-heading: var(--font-fraunces), 'Fraunces', Georgia, serif;
      --font-body: var(--font-inter), 'Inter', system-ui, sans-serif;
      --font-mono: var(--font-ibm-plex-mono), 'IBM Plex Mono', monospace;

      /* shadcn/ui compatibility tokens */
      --background: var(--bg-page);
      --foreground: var(--text-primary);
      --card: var(--bg-card);
      --card-foreground: var(--text-primary);
      --popover: var(--bg-card);
      --popover-foreground: var(--text-primary);
      --primary-foreground: #111111;
      --secondary: #E7E8E5;
      --secondary-foreground: #111111;
      --muted: #F2F3F0;
      --muted-foreground: #666666;
      --accent-foreground: #111111;
      --destructive-foreground: #FFFFFF;
      --input: #E8E4DF;
      --ring: #7C9082;
      --radius: 16px;
    }

    /* ===== Tailwind v4 Theme ===== */
    @theme inline {
      --color-bg-page: var(--bg-page);
      --color-bg-card: var(--bg-card);
      --color-bg-muted: var(--bg-muted);
      --color-bg-surface: var(--bg-surface);

      --color-warm-accent: var(--warm-accent);
      --color-warm-accent-tint: var(--warm-accent-tint);
      --color-accent: var(--accent);
      --color-accent-tint: var(--accent-tint);
      --color-primary: var(--primary);

      --color-text-primary: var(--text-primary);
      --color-text-body: var(--text-body);
      --color-text-secondary: var(--text-secondary);
      --color-text-muted: var(--text-muted);

      --color-border: var(--border);
      --color-border-subtle: var(--border-subtle);

      --color-success: var(--success);
      --color-error: var(--error);
      --color-destructive: var(--destructive);

      --radius-card: var(--radius-card);
      --radius-button: var(--radius-button);
      --radius-input: var(--radius-input);
      --radius-badge: var(--radius-badge);
      --radius-pill: var(--radius-pill);
      --radius-small: var(--radius-small);

      --font-heading: var(--font-heading);
      --font-body: var(--font-body);
      --font-mono: var(--font-mono);
    }

    /* ===== Base Styles ===== */
    body {
      background: var(--bg-page);
      color: var(--text-body);
      font-family: var(--font-body);
    }

    h1, h2, h3, h4, h5, h6 {
      font-family: var(--font-heading);
      color: var(--text-primary);
    }
    ```

    **Key mapping for component usage:**
    - `bg-bg-page` → page background (#FAF8F5)
    - `bg-bg-card` → card background (#FFFFFF)
    - `text-text-primary` → heading text (#2D2D2D)
    - `text-warm-accent` → warm accent text (#C4956A)
    - `border-border` → card borders (#E8E4DF)
    - `rounded-[--radius-card]` → card border radius (20px)
    - `font-heading` → Fraunces serif
    - `font-mono` → IBM Plex Mono for prices

    NOTE: shadcn/ui init may have added its own CSS variables. Merge carefully — keep shadcn compatibility tokens (--background, --foreground, --card, etc.) but override their values with our design tokens. The shadcn variables should reference our design tokens.
