<task type="auto">
  <name>Task 1: Page fade-in transition (PLSH-V2-01)</name>
  <files>components/page-transition.tsx, app/(main)/layout.tsx, app/globals.css</files>
  <action>
Create a CSS-based page transition that fades in content when navigating between routes. Since Next.js App Router re-mounts page components on navigation, a CSS animation on mount achieves the fade effect.

**1. Add keyframe animation to globals.css**

Add this at the end of `app/globals.css`:

```css
@keyframes page-fade-in {
  from {
    opacity: 0;
    transform: translateY(4px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.animate-page-in {
  animation: page-fade-in 0.3s ease-out both;
}
```

**2. Create components/page-transition.tsx**

A simple client component that wraps children with the fade-in animation class. Uses `usePathname()` from `next/navigation` as a key to re-trigger animation on route changes:

```tsx
'use client';

import { usePathname } from 'next/navigation';

export function PageTransition({ children }: { children: React.ReactNode }) {
  const pathname = usePathname();

  return (
    <div key={pathname} className="animate-page-in">
      {children}
    </div>
  );
}
```

**3. Wrap children in app/(main)/layout.tsx**

Import `PageTransition` from `@/components/page-transition` and wrap `{children}` inside `<main>`:

```tsx
import { PageTransition } from '@/components/page-transition';

// Inside the return, replace {children} with:
<PageTransition>{children}</PageTransition>
```

Keep the `<main>` wrapper and all other elements unchanged. Only wrap {children} inside PageTransition.

Also apply the same pattern to the auth layout if one exists at `app/auth/login/page.tsx` — but do NOT create a new layout for auth. The login page is standalone, so just add `className="animate-page-in"` to its outermost wrapper div.
