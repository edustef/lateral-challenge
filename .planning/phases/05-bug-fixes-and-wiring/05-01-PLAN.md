Three changes to fix AUTH-03 and CHKT-08:

```typescript
export default async function middleware(request: NextRequest) {
```
Everything else in the file stays the same. The `lib/supabase/proxy.ts` helper file is NOT renamed (only the root entry point).

**2. Update login page to read and forward ?redirect= param (CHKT-08)**

In `app/auth/login/page.tsx`:
- Import `useSearchParams` from `next/navigation`
- Read the `redirect` search param: `const searchParams = useSearchParams(); const redirectTo = searchParams.get('redirect')`
- Pass `redirectTo` to the `loginWithOtp` call by appending it to the FormData before calling the action: `formData.append('redirect', redirectTo ?? '')`
- Wrap the component content in a `<Suspense>` boundary since `useSearchParams()` requires it in Next.js App Router, OR extract the form into a child component wrapped with Suspense. Simplest approach: extract a `LoginForm` component that uses `useSearchParams`, and wrap it in `<Suspense>` in the page component.

**3. Update loginWithOtp to forward redirect through emailRedirectTo (CHKT-08)**

In `lib/actions/auth.ts`:
- Read `redirect` from formData: `const redirectPath = formData.get('redirect') as string | null`
- Build the emailRedirectTo URL to include the redirect param: if `redirectPath` is truthy, append `?redirect=${encodeURIComponent(redirectPath)}` to the callback URL
- Result: `emailRedirectTo: \`\${baseUrl}/auth/callback\${redirectPath ? \`?redirect=\${encodeURIComponent(redirectPath)}\` : ''}\``

The auth callback at `app/auth/callback/route.ts` already reads `?redirect=` and redirects there — no changes needed to that file.

**Full redirect chain after fix:**
1. Unauthenticated user visits `/stays/treehouse/book`
2. `middleware.ts` (via `updateSession`) redirects to `/auth/login?redirect=/stays/treehouse/book`
3. Login page reads `?redirect=/stays/treehouse/book` and passes it through form
4. `loginWithOtp` sets `emailRedirectTo` to `/auth/callback?redirect=%2Fstays%2Ftreehouse%2Fbook`
5. After OTP verification, Supabase redirects to `/auth/callback?code=xxx&redirect=%2Fstays%2Ftreehouse%2Fbook`
6. Callback exchanges code, reads `redirect` param, redirects to `/stays/treehouse/book`
