**Accessibility improvements (PLSH-05):**

1. **components/vibe-picker.tsx** — Add aria-labels and keyboard nav:
   - Each chip button: `aria-label="Filter by {label}"` and `aria-pressed={isSelected}`
   - Stepper steps should use `role="group"` with `aria-label="Step {n}: {title}"`

2. **components/vibe-picker-mobile.tsx** — Accessibility on mobile picker:
   - Trigger button: `aria-label="Open vibe filter"`, `aria-expanded={open}`
   - Drawer/sheet: `aria-label="Vibe filter"`

3. **components/header.tsx** — Navigation aria:
   - `<nav aria-label="Main navigation">` wrapper
   - Logo link: `aria-label="Lateral — home"`
   - Ensure skip-to-content is NOT needed (header is minimal)

4. **components/checkout-form.tsx** — Form accessibility:
   - All input fields need associated `<label>` elements (or aria-label if labels are visual)
   - Step indicator: `aria-label="Checkout step {n} of 4: {stepName}"` on each step
   - Submit button: aria-busy during submission

5. **components/review-form.tsx** — Form accessibility:
   - Star rating: use `role="radiogroup"` with `aria-label="Rating"`, each star is `role="radio"` with `aria-checked` and `aria-label="{n} star{s}"`
   - Textarea: `aria-label="Review comment"`

6. **components/booking-sidebar.tsx** — Interactive elements:
   - Date inputs: ensure proper labels or aria-labels
   - Guest counter: `aria-label="Number of guests"`
   - Book button: `aria-label="Book this stay"`

7. **Global focus rings** — Verify that interactive elements show visible focus indicators. The existing Tailwind setup likely handles this via `focus-visible:ring-2 focus-visible:ring-accent`. If any buttons or links are missing focus styles, add them. Check the button.tsx component and any custom buttons.

**Structured server action logging (PLSH-06):**

Add logging to every server action (NOT data fetching functions like getStays — only mutating actions and key queries):

For each server action, wrap the body with:
```typescript
const start = performance.now();
const actionName = 'createBooking'; // or whatever
console.log(`[action] ${actionName} start`, { stayId, checkIn, checkOut, guests });
try {
  // ... existing logic ...
  console.log(`[action] ${actionName} ok`, { duration: Math.round(performance.now() - start) + 'ms' });
  return result;
} catch (err) {
  console.error(`[action] ${actionName} error`, { duration: Math.round(performance.now() - start) + 'ms', error: err });
  throw err;
}
```

Apply to:
- `lib/actions/bookings.ts` — `createBooking`: log stayId, checkIn, checkOut, guests (not contact PII)
- `lib/actions/reviews.ts` — `createReview`: log stayId, rating (not comment content)
- `lib/actions/stays.ts` — `getStays`: log filters object. `getStayBySlug`: log slug. Do NOT log searchStaysPreview or getReviewsForStay (too noisy).

Keep the logging lightweight — no external dependencies, just console.log/console.error with a `[action]` prefix.
