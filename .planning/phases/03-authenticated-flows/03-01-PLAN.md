1. **lib/supabase/proxy.ts** — Create the Supabase auth proxy (NOT proxy.ts (root) — per user decision, this is proxy.ts):
   - Export an `updateSession(request: NextRequest)` function
   - Use `createServerClient` from `@supabase/ssr` with `NextRequest`/`NextResponse` cookie handling
   - Read cookies from request, write cookies to response using `request.cookies.get`/`response.cookies.set` pattern
   - Call `supabase.auth.getUser()` to refresh the session token (this is what keeps sessions alive)
   - If user is NOT authenticated and path starts with `/stays/` and ends with `/book`, redirect to `/auth/login?redirect={encodeURIComponent(request.nextUrl.pathname)}`
   - Return the response with updated cookies
   - Then create a `proxy.ts (root)` at the project ROOT that imports and calls `updateSession` from proxy.ts. Export a `config` matcher that excludes static files: `matcher: ['/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)']`

2. **lib/actions/auth.ts** — Server actions for auth:
   - `loginWithOtp(formData: FormData)`: Extract email from formData, call `supabase.auth.signInWithOtp({ email, options: { emailRedirectTo: '{origin}/auth/callback' } })`. Return `{ error?: string; success?: boolean }`. Use `headers()` from `next/headers` to get the origin for redirect URL.
   - `signOut()`: Call `supabase.auth.signOut()`, then `redirect('/')` from `next/navigation`.

3. **app/auth/login/page.tsx** — Login page (server component with client form):
   - Accept `searchParams` prop with optional `redirect` param
   - Show a centered card with "Sign in to Lateral" heading
   - Email input field + "Send magic link" submit button
   - Use a client component `LoginForm` (can be in same file with 'use client' boundary) that:
     - Calls `loginWithOtp` server action on submit
     - Shows loading state on button while submitting (useTransition or useActionState)
     - On success: show "Check your email for a magic link!" message
     - On error: show error message
   - Style with existing design tokens (bg-bg-card, border-border, rounded-card, accent button)
   - This page lives OUTSIDE the (main) route group — no header needed on auth pages

4. **app/auth/callback/route.ts** — Auth callback handler:
   - Export `GET` handler
   - Extract `code` from `request.nextUrl.searchParams`
   - If code exists, exchange it: `supabase.auth.exchangeCodeForSession(code)`
   - Extract `redirect` param from searchParams, default to '/'
   - Redirect to the `redirect` param (or '/' if not provided) using `NextResponse.redirect`
   - Handle errors by redirecting to `/auth/login?error=auth_failed`
